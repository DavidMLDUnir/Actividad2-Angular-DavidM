2. Instalación de Angular

¿Se precisa instalación?

Sí. Para trabajar con Angular es necesario instalar el entorno base que permite usar Angular CLI, la herramienta oficial para crear, compilar y ejecutar proyectos de Angular. Esta instalación se realiza en el equipo local, aunque también existen versiones online como StackBlitz para pruebas rápidas.

¿Qué pasos hay que realizar para la instalación?

Para instalar Angular en tu equipo hay que seguir estos pasos:

  1. Instalar Node.js Angular necesita Node.js para gestionar dependencias a través de npm (Node Package Manager). Se debe descargar desde la página oficial y seguir el instalador. A continuación, adjunto el enlace a la página oficial para la descarga Node.js
  2. Verificar la instalación Ejecutamos en nuestra terminal: 
    node -v  
npm -v
  3. Instalar Angular CLI Con el siguiente comando: 
    npm install -g @angular/cli

¿Versiones necesarias?

Para trabajar con Angular 21, la versión recomendada de Node.js suele ser la más reciente que nos aparecera con el indicativo LTS (Long Term Support), habitualmente la versión 18 o 20.

Puedes comprobar la compatibilidad en la documentación oficial, pero la combinación más usada es:

  • Angular CLI: 17+ o 21 (según la instalación)
  • Node.js: 18.x o 20.x LTS
  • npm: 9 o superior

¿Rutas para tener en cuenta?

No es necesario modificar rutas del sistema, pero al crear el proyecto se recomienda hacerlo en una carpeta sin espacios para evitar problemas al ejecutar comandos.

La ruta del proyecto será la carpeta donde se ejecute ng new.

¿Variables de entorno?

Angular no requiere variables de entorno específicas para funcionar, pero en ciertos proyectos avanzados pueden configurarse mediante archivos como: src/environments/environment.ts.

Para la instalación básica, no es necesario configurar nada.

¿Necesita base de datos?

Angular no necesita base de datos para instalarse ni para funcionar en su entorno de desarrollo. Sin embargo, puede conectarse a APIs REST, servicios web o backends externos que sí usen bases de datos (por ejemplo Node.js, Java Spring, Firebase, etc.).
